Arpadroid - Themes

Arpadroid Themes package helps keeping your styles organized into different theme stylesheets, providing high scalability, maintainability and performance, ultimately striving to boost CSS development productivity and efficiency. It can be integrated to any build system, toolchain, framework or application. Supports LESS and SCSS, as well as minification.

Try it yourself!

Configuration

Arpadroid Themes is basically a stylesheet compiler. It leverages ability to create multiple stylesheets, which are compiled from many different stylesheet sources across your application.

Getting started is easy:

  1. The theme styles will be kept under a directory within your application source code with a name and location of your own choosing. The name of this directory will represent the theme stylesheet name e.g 'default', 'dark', 'light', etc...

    It is required that you have a JSON theme configuration file in the root of this directory. The naming convention is '[directoryName].config.json' e.g 'default.config.json', 'dark.config.json', 'light.config.json', etc...

    The configuration file should define the theme stylesheets inside the includes array. The stylesheet includes are relative to the theme directory and the extension should be omitted. e.g. 

                                            
                                            {
                                                "includes": [
                                                    "vars/colors",
                                                    "vars/easing",
                                                    "vars/layout",
                                                    "vars/screen",
                                                    "vars/typography",
                                                    "vars/animations",
                                                    "vars/variables",
                                                    "main/fonts",
                                                    "main/main",
                                                    "components/headings",
                                                    "components/lists",
                                                    "components/scrollbar",
                                                    "components/summary",
                                                    "components/link",
                                                    "components/list"
                                                ]
                                            }

    It is possible to have theme stylesheets outside the theme directory, which are compiled back to the output theme stylesheet, by convention these stylesheets will have a sub-extension matching the theme name e.g. 'buttonComponent.dark.css' continue reading to find all about it..
