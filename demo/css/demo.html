Arpadroid - Stylesheet Bundler

Welcome to this demo of Arpadroid Stylesheet Bundler.
Refer to the link above to access the source code and documentation on GitHub.

You can use this as a playground to experiment with the theme stylesheets and explore the possibilities. I will demonstrate some handy use cases below.

Setting up

To get the demo environment set up, you can follow these steps:

  1. If you have checked out the project from GitHub, you will need to run npm install at the root of the project. If you have installed it through NPM then you don't need to do this.
  2. To get the bundler running in this demo you can go to the root of this project using the CLI and run the command npm run bundle:css:dev.
    This should be watching your files and recompiling them on change.
  3. To get Livereload working also run the command guard from the root of this project.
    Make sure you have read the docs and installed the necessary dependencies.
    If done correctly, you should see the new styles instantly refresh in the browser when you make a change and the state of your page will remain intact.

Toggling themes interactively

We have added a stylesheet in the head of this document for the dark theme. If you inspect the source you will notice that the dark theme stylesheet is disabled by default. It has an id of dark-theme and we can use JavaScript to toggle it on and off.

Although we are using a custom element as the button and it is not immediately obvious what's happening, all we are doing is toggling the disabled attribute on the stylesheet link. This is a simple way to toggle the theme on and off.

Toggling themes based on screen size

If you find yourself battling with convoluted media queries and complex styles to switch between mobile and desktop, possibly because the styles in mobile happen to be very different to those of large screens, then you might find this approach useful.

In the following way, we can toggle the desktop/mobile themes whenever appropriate.

<link id="mobile-theme" rel="stylesheet" media="screen and (max-width: 700px)" href="themes/mobile/mobile.bundled.css" />
<link id="desktop-theme" rel="stylesheet" media="screen and (min-width: 701px)" href="themes/desktop/desktop.bundled.css" />

This is a simple way to toggle the theme based on the screen size. You can see the difference by resizing the browser window.

Did you know? Toggling stylesheets off when not needed can help reduce the load on the browser and improve rendering performance in a considerable way.

Last but not least... organizing your stylesheets in the way Arpadroid Themes allows to, helps reduce the complexity and cognitive load of working with large stylesheets, keeping your styles more organized and maintainable.
